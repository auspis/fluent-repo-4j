# Advanced Usage Examples

This document provides complete, copy-paste-ready examples for common scenarios.

## 1. Application-Provided UUID IDs

Useful for domains where you generate IDs on the client side (e.g., CQRS, event sourcing).

```java
import jakarta.persistence.*;
import java.util.UUID;

@Table(name = "events")
public class Event {
    @Id
    private UUID id;
    
    private String description;
    
    private LocalDateTime createdAt;
    
    public Event() {
    }
    
    public Event(String description) {
        this.id = UUID.randomUUID();
        this.description = description;
        this.createdAt = LocalDateTime.now();
    }
    
    // Getters and setters
    public UUID getId() { return id; }
    public void setId(UUID id) { this.id = id; }
    public String getDescription() { return description; }
    public void setDescription(String description) { this.description = description; }
    public LocalDateTime getCreatedAt() { return createdAt; }
    public void setCreatedAt(LocalDateTime createdAt) { this.createdAt = createdAt; }
}
```

**Repository**:
```java
import org.springframework.data.repository.CrudRepository;

public interface EventRepository extends CrudRepository<Event, UUID> {
}
```

**Usage**:
```java
@Service
public class EventService {
    
    private final EventRepository eventRepository;
    
    public EventService(EventRepository eventRepository) {
        this.eventRepository = eventRepository;
    }
    
    @Transactional
    public Event publishEvent(String description) {
        Event event = new Event(description);  // UUID auto-generated in constructor
        return eventRepository.save(event);     // INSERT with UUID as ID
    }
    
    public Event getEvent(UUID id) {
        return eventRepository.findById(id).orElseThrow();
    }
}
```

## 2. Database Auto-Increment IDs (Identity Strategy)

Standard approach for sequential IDs generated by the database.

```java
import jakarta.persistence.*;

@Table(name = "users")
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String username;
    private String email;
    
    public User() {
    }
    
    public User(String username, String email) {
        this.username = username;
        this.email = email;
    }
    
    // Getters and setters
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    public String getUsername() { return username; }
    public void setUsername(String username) { this.username = username; }
    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }
}
```

**Repository**:
```java
public interface UserRepository extends CrudRepository<User, Long> {
}
```

**Usage**:
```java
@Service
public class UserService {
    
    private final UserRepository userRepository;
    
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
    
    @Transactional
    public User registerUser(String username, String email) {
        User user = new User(username, email);  // id = null
        return userRepository.save(user);        // INSERT, database returns generated ID
    }
}
```

## 3. Custom isNew() Logic with Persistable<ID>

Advanced: control exactly when an entity is considered "new" vs "existing".

```java
import jakarta.persistence.*;
import org.springframework.data.domain.Persistable;

@Table(name = "orders")
public class Order implements Persistable<UUID> {
    @Id
    private UUID id;
    
    private String orderNumber;
    private LocalDateTime orderDate;
    
    @Transient
    private boolean isNew = true;
    
    public Order() {
        this.id = UUID.randomUUID();
    }
    
    public Order(String orderNumber) {
        this();
        this.orderNumber = orderNumber;
        this.orderDate = LocalDateTime.now();
    }
    
    @Override
    public UUID getId() {
        return id;
    }
    
    @Override
    public boolean isNew() {
        return isNew;
    }
    
    @PostLoad
    @PostPersist
    public void markPersisted() {
        this.isNew = false;
    }
    
    // Getters and setters
    public String getOrderNumber() { return orderNumber; }
    public void setOrderNumber(String orderNumber) { this.orderNumber = orderNumber; }
    public LocalDateTime getOrderDate() { return orderDate; }
    public void setOrderDate(LocalDateTime orderDate) { this.orderDate = orderDate; }
}
```

**Repository** (same interface):
```java
public interface OrderRepository extends CrudRepository<Order, UUID> {
}
```

**Usage**:
```java
@Service
public class OrderService {
    
    private final OrderRepository orderRepository;
    
    public OrderService(OrderRepository orderRepository) {
        this.orderRepository = orderRepository;
    }
    
    @Transactional
    public Order createOrder(String orderNumber) {
        Order order = new Order(orderNumber);
        return orderRepository.save(order);  // isNew() returns true → INSERT
    }
    
    @Transactional
    public Order updateOrder(UUID id, String newOrderNumber) {
        Order order = orderRepository.findById(id).orElseThrow();
        // At this point, order.isNew() returns false (set by @PostLoad)
        order.setOrderNumber(newOrderNumber);
        return orderRepository.save(order);  // isNew() returns false → UPDATE
    }
}
```

## 4. Transaction with Multiple Repository Operations

Demonstrate ACID semantics with multiple calls wrapped in a transaction.

```java
@Service
public class TransferService {
    
    private final UserRepository userRepository;
    private final TransactionRepository transactionRepository;
    
    public TransferService(UserRepository userRepository, 
                           TransactionRepository transactionRepository) {
        this.userRepository = userRepository;
        this.transactionRepository = transactionRepository;
    }
    
    @Transactional
    public void transferBalance(Long fromUserId, Long toUserId, BigDecimal amount) {
        // Fetch users within the same transaction (they see the same state)
        User fromUser = userRepository.findById(fromUserId).orElseThrow();
        User toUser = userRepository.findById(toUserId).orElseThrow();
        
        // Validate business logic
        if (fromUser.getBalance().compareTo(amount) < 0) {
            throw new IllegalArgumentException("Insufficient balance");
        }
        
        // Update both users
        fromUser.setBalance(fromUser.getBalance().subtract(amount));
        toUser.setBalance(toUser.getBalance().add(amount));
        
        userRepository.save(fromUser);
        userRepository.save(toUser);
        
        // Record the transaction
        Transaction txn = new Transaction(fromUserId, toUserId, amount);
        transactionRepository.save(txn);
        
        // All changes are committed atomically; if any operation throws, the entire
        // transaction is rolled back.
    }
}
```

## 5. Filtering Results in Application Code

Since custom query methods are not yet supported, you can fetch all and filter in application code.

```java
@Service
public class UserSearchService {
    
    private final UserRepository userRepository;
    
    public UserSearchService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
    
    public List<User> findUsersByEmail(String emailPattern) {
        return ((List<User>) userRepository.findAll())
            .stream()
            .filter(user -> user.getEmail().contains(emailPattern))
            .collect(Collectors.toList());
    }
    
    public User findByUsername(String username) {
        return ((List<User>) userRepository.findAll())
            .stream()
            .filter(user -> user.getUsername().equals(username))
            .findFirst()
            .orElse(null);
    }
}
```

**Note**: For production with large tables, implement using fluent-sql-4j directly, or wait for future versions with PartTree support.

## 6. Handling Different Column Names

## 7. Excluding Fields from Persistence

    @Transient
    private boolean isLoggedIn;       // Computed at runtime, not stored

## 8. Batch Operations

Insert or delete multiple entities. Note: executed as loops, not as true batch statements (yet).

```java
@Service
public class BulkUserService {
    
    private final UserRepository userRepository;
    
    public BulkUserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
    
    @Transactional
    public void importUsers(List<User> users) {
        for (User user : users) {
            userRepository.save(user);  // Individual INSERT per user
        }
        // All inserts are atomic; if any fails, entire batch rolls back
    }
    
    @Transactional
    public void deleteUsers(List<Long> userIds) {
        for (Long id : userIds) {
            userRepository.deleteById(id);  // Individual DELETE per ID
        }
    }
}
```

**Performance note**: For high-volume bulk operations (1000s of rows), consider using fluent-sql-4j's batch API directly for better performance.

## 9. Error Handling & Validation

Best practices for error handling in repository operations.

```java
@Service
public class ErrorHandlingService {
    
    private final UserRepository userRepository;
    private static final Logger logger = LoggerFactory.getLogger(ErrorHandlingService.class);
    
    public ErrorHandlingService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
    
    public Optional<User> safeGetUser(Long id) {
        try {
            return userRepository.findById(id);
        } catch (org.springframework.dao.DataAccessException e) {
            logger.error("Database error while fetching user {}", id, e);
            // Spring has already translated SQLException to DataAccessException
            return Optional.empty();
        }
    }
    
    @Transactional
    public User createUserWithValidation(String username, String email) {
        // Validate input
        if (username == null || username.isEmpty()) {
            throw new IllegalArgumentException("Username cannot be empty");
        }
        if (email == null || !email.contains("@")) {
            throw new IllegalArgumentException("Invalid email format");
        }
        
        // Check for duplicates (ANTI-PATTERN: refactor to custom query when available)
        boolean exists = ((List<User>) userRepository.findAll())
            .stream()
            .anyMatch(u -> u.getUsername().equals(username));
        
        if (exists) {
            throw new IllegalArgumentException("Username already exists");
        }
        
        User user = new User(username, email);
        return userRepository.save(user);
    }
}
```

## 10. Testing with Spring Context

Complete test example using Spring Boot Test and H2 in-memory database.

```java
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.autoconfigure.jdbc.AutoConfigureTestDatabase;
import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.transaction.annotation.Transactional;

import static org.junit.jupiter.api.Assertions.*;

@SpringBootTest
@AutoConfigureTestDatabase(replace = AutoConfigureTestDatabase.Replace.ANY)
public class UserRepositoryIntegrationTest {
    
    @Autowired
    private UserRepository userRepository;
    
    @Test
    @Transactional
    public void testCreateAndRetrieveUser() {
        // Arrange
        User user = new User("john_doe", "john@example.com");
        
        // Act
        User saved = userRepository.save(user);
        Long userId = saved.getId();
        
        // Assert
        assertTrue(userId != null, "ID should be generated");
        
        User retrieved = userRepository.findById(userId).orElseThrow();
        assertEquals("john_doe", retrieved.getUsername());
        assertEquals("john@example.com", retrieved.getEmail());
    }
    
    @Test
    @Transactional
    public void testUpdateUser() {
        // Arrange
        User user = new User("alice", "alice@example.com");
        User saved = userRepository.save(user);
        Long userId = saved.getId();
        
        // Act
        User toUpdate = userRepository.findById(userId).orElseThrow();
        toUpdate.setUsername("alice_updated");
        userRepository.save(toUpdate);
        
        // Assert
        User updated = userRepository.findById(userId).orElseThrow();
        assertEquals("alice_updated", updated.getUsername());
    }
    
    @Test
    @Transactional
    public void testDeleteUser() {
        // Arrange
        User user = new User("bob", "bob@example.com");
        User saved = userRepository.save(user);
        Long userId = saved.getId();
        
        assertTrue(userRepository.existsById(userId));
        
        // Act
        userRepository.deleteById(userId);
        
        // Assert
        assertFalse(userRepository.existsById(userId));
    }
    
    @Test
    @Transactional
    public void testFindAll() {
        // Arrange
        userRepository.save(new User("user1", "user1@example.com"));
        userRepository.save(new User("user2", "user2@example.com"));
        userRepository.save(new User("user3", "user3@example.com"));
        
        // Act
        List<User> all = (List<User>) userRepository.findAll();
        
        // Assert
        assertEquals(3, all.size());
    }
    
    @Test
    @Transactional
    public void testCount() {
        // Arrange
        userRepository.save(new User("user1", "user1@example.com"));
        userRepository.save(new User("user2", "user2@example.com"));
        
        // Act
        long count = userRepository.count();
        
        // Assert
        assertEquals(2, count);
    }
}
```

## 11. Using Persistable Interface for Complex Scenarios

Example: suppress `@Transient` field mutations that should trigger updates.

```java
import jakarta.persistence.*;
import org.springframework.data.domain.Persistable;

@Table(name = "versioned_documents")
public class Document implements Persistable<UUID> {
    @Id
    private UUID id;
    
    private String title;
    private String content;
    private int version;
    
    @Transient
    private boolean isNew = true;
    
    @Transient
    private LocalDateTime lastAccessTime;  // Should NOT trigger update
    
    public Document() {
        this.id = UUID.randomUUID();
    }
    
    public Document(String title, String content) {
        this();
        this.title = title;
        this.content = content;
        this.version = 1;
    }
    
    @Override
    public UUID getId() {
        return id;
    }
    
    @Override
    public boolean isNew() {
        return isNew;
    }
    
    @PostLoad
    @PostPersist
    public void markPersisted() {
        this.isNew = false;
    }
    
    public void accessTime() {
        this.lastAccessTime = LocalDateTime.now();
        // This does NOT modify the "isNew" flag, so save() will UPDATE correctly
    }
    
    // Getters and setters...
}
```

## 12. Configuration Best Practices

```java
import org.springframework.boot.autoconfigure.EnableAutoConfiguration;
import org.springframework.context.annotation.Configuration;
import org.springframework.data.jdbc.repository.config.EnableJdbcRepositories;

@Configuration
@EnableAutoConfiguration
@EnableFluentRepositories(basePackages = "com.example.repositories")
public class DataAccessConfig {
    // No additional configuration needed!
    // DataSource is auto-configured from application.yml
    // Dialect is auto-detected from DataSource metadata
}
```

In `application.yml`:

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/myapp
    username: postgres
    password: password
    driver-class-name: org.postgresql.Driver

# Optional: enable SQL logging for debugging
logging:
  level:
    org.springframework.jdbc: DEBUG
    org.springframework.transaction: DEBUG
```
